Methodology

This application utilizes a Logistic Regression model trained on a dataset of student health and academic factors to predict the likelihood of depression.

Key Steps in Model Development:

  1. Data Collection: Data was gathered from various sources, including student demographics, academic performance indicators (like CGPA), sleep patterns, dietary habits, and self-reported mental health indicators.
  2. Data Preprocessing:
    • Handling missing values.
    • Encoding categorical features (e.g., Gender, Degree, Sleep Duration categories, Suicidal thoughts, Family History, Financial Stress) using `LabelEncoder`.
    • Converting relevant columns to appropriate numeric types.
  3. Feature Selection: Relevant features were identified and selected for model training.
  4. Model Training: A Logistic Regression model was trained on the preprocessed dataset.
  5. Model Evaluation: The model's performance was evaluated using metrics such as accuracy and confusion matrix.
  6. Model Serialization: The trained model and necessary encoders (`gender-encoder.pkl`, `degree-encoder.pkl`, `depression-lr-model.pkl`) were saved using Python's `pickle` module for deployment.

The web application then loads these saved models and encoders to perform real-time predictions based on user input.

For detailed analysis and exploratory data visualization, refer to the original Jupyter Notebook (`students_depression.ipynb`).
